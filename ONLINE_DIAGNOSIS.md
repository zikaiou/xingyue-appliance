# 线上问题诊断说明

当前网站已经包含 Brand Film、ST-812、12 个产品和 Factory 等主要模块。线上仅通过公开页面无法直接读取浏览器运行后的全部 computed style，因此不能在没有 DevTools 检查的情况下断言唯一原因。

根据“Hero 与 Factory 最明显”的表现，优先级如下：

1. **文字祖先节点残留 transform**  
   Hero 或 Factory 入场动画结束后仍存在 matrix/translate/scale，浏览器继续把整个内容区作为合成图层栅格化。

2. **区块级动画作用范围过大**  
   对整个 section 或内容 wrapper 做淡入、缩放、视差，会让内部所有文字一起进入 GPU 合成层。

3. **Factory 图片运动与文字共用父容器**  
   为了做镜头移动，若移动的是整个 Factory 容器而非图片层，文字也会被持续重采样。

4. **字体真实字重缺失**  
   页面指定 500/600，但只加载 400/700 时，浏览器可能合成中间字重，视觉会偏软。

5. **非整数缩放或浏览器缩放**  
   `scale(.98)`、小数定位、Windows 非 100% 浏览器缩放会放大模糊现象。

实施时先处理 1–3，再检查字体文件和字重。不要仅靠 `font-smoothing` 掩盖问题。
